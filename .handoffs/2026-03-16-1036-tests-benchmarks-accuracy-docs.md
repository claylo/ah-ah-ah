## Decisions made

- **Fast-path heuristic for MarkdownDecomposer** — added `has_table_separator()` that scans for lines where content after `|` is only `-:| \t` with at least one dash. Three-tier rejection cascade: `!contains('|')` → `!has_table_separator()` → `find_table_ranges().is_empty()`. Avoids pulldown-cmark on code with pipes.
- **API smoke test uses Claude Code CLI** — not `curl` or the Python SDK. Requires `ANTHROPIC_API_KEY="" CLAUDECODE="" DISABLE_PROMPT_CACHING=1` plus `--system-prompt "Reply with OK." --tools "" --disable-slash-commands --plugin-dir no-plugins --settings '{}' --setting-sources local` to strip overhead. A baseline run measures the ~574-token constant overhead and subtracts it.
- **Three assertion types in fixture tests** — `assert_exact` (7 strings where delta=0), `assert_overcount` (3 strings: CJK, emoji, Latin — must be >= API), `assert_near` (8 strings with tolerance bands for known merge-order divergence).

## Landmines

- **Nested `claude` CLI auth** — when running from inside a Claude Code session, you MUST set `ANTHROPIC_API_KEY=""` to clear the parent's internal session token (24 chars, not a real API key). Without this, the subprocess gets "Invalid API key". Also set `CLAUDECODE=""` to avoid nested session detection. This broke recently and was diagnosed this session.
- **`--system-prompt ""` does NOT strip the system prompt** — empty string is ignored. Use `--system-prompt "Reply with OK."` or similar minimal text. Even then, ~574 tokens of environment context (cwd, git, OS) are injected. The smoke test script measures and subtracts this baseline.
- **`claude_vocab.json` is 475KB embedded via `include_str!`** — the Aho-Corasick automaton is built lazily (`LazyLock`). First call has initialization cost (~77ms in benchmarks).
- **bpe-openai 0.3 build is slow** — deserializes gzipped tiktoken data at compile time. First `cargo build` takes ~60s. Subsequent builds cached.
- **Fixture test strings with `\n`** — the bash script sends literal `\n` (two chars), not real newlines. The Rust tests must use `\\n` to match. See `fixture_rust_if_let` which was fixed for this.
- **`no-plugins/` directory** — needed by the smoke test script. Created manually with a skeleton `installed_plugins.json` and `known_marketplaces.json`. Gitignored.
- **`tests/fixtures/*.json`** — raw API responses from smoke test runs. Gitignored (regenerated by the script). Only `.gitkeep` is tracked.
